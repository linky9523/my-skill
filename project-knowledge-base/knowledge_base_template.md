# 项目知识库使用模板

本文档用于说明团队如何放置项目资料、如何命名文件、如何维护版本，以及新增资料时需要记录什么。

## 1. 项目资料怎么放

每个项目使用一个独立文件夹：

```text
knowledge_base/projects/{project_name}/
```

项目文件夹内建议包含：

```text
project_index.md
01_product_info/
02_selling_points/
03_competitors/
04_user_research/
05_marketing_outputs/
06_meeting_notes/
99_archive/
```

`project_index.md` 是项目总索引，记录项目背景、定位、核心卖点、关键竞品、主要资料和待补充信息。每次新增重要资料后，建议同步更新该文件。

## 2. 命名规则

建议文件名包含日期、主题、版本或状态：

```text
YYYY-MM-DD_资料主题_版本.扩展名
```

示例：

```text
2026-05-29_Find_X10e_产品参数表_v1.xlsx
2026-05-29_Find_X10e_核心卖点梳理_v1.md
2026-05-29_Find_X10e_竞品价格配置对比_v1.xlsx
2026-05-29_Find_X10e_官网框架初稿_v1.docx
2026-05-29_Find_X10e_评审会议纪要_v1.md
```

命名建议：

- 日期使用资料形成或确认日期，不确定时使用放入知识库的日期。
- 主题尽量明确，避免只写 `资料1`、`新版`、`最终版`。
- 版本建议使用 `v1`、`v2`、`v3`，不要只依赖“最新”“最终”等字样。
- 已废弃或过期文件移动到 `99_archive/`，并在文件名或索引中说明原因。

## 3. 文件夹说明

### `01_product_info`

放产品基础信息、定位、参数、版本、价格、上市节奏等。

适合放：

- 产品参数表
- PRD 摘要
- 版本和 SKU 信息
- 价格和上市节奏
- 产品定位说明

### `02_selling_points`

放卖点信息表、功能说明、用户收益、技术解释、传播表达等。

适合放：

- 卖点总表
- KSP/KV 说明
- 技术能力解释
- 用户收益表达
- 官网或发布会卖点口径

### `03_competitors`

放竞品资料、竞品卖点、价格配置对比、传播打法分析等。

适合放：

- 竞品参数表
- 竞品价格配置对比
- 竞品官网或发布会拆解
- 竞品传播表达分析

### `04_user_research`

放用户调研、访谈纪要、评论分析、目标人群洞察等。

适合放：

- 用户访谈纪要
- 评论和舆情分析
- 目标人群画像
- 使用场景洞察

### `05_marketing_outputs`

放官网框架、PPT、发布会文案、门店话术、社媒内容、培训材料等。

适合放：

- 官网框架
- PPT 或演示稿
- 发布会文案
- 门店培训话术
- 社媒内容草稿
- 历史交付件

### `06_meeting_notes`

放会议纪要、评审意见、关键决策、修改记录等。

适合放：

- 项目例会纪要
- 评审反馈
- 修改记录
- 决策记录

### `99_archive`

放过期资料、旧版本文件、已废弃方案等。

适合放：

- 旧版参数表
- 已废弃官网框架
- 过期竞品资料
- 被替换的会议纪要或方案

## 4. 版本管理建议

- 重要资料保留历史版本，不直接覆盖。
- 最新版本在 `project_index.md` 中标记清楚。
- 旧版本移动到 `99_archive/`，并说明归档原因。
- 如果多个资料口径冲突，在 `project_index.md` 中新增“资料冲突记录”。
- 涉及参数、价格、上市时间、合规表述的资料，需要标明来源和确认状态。

## 5. 新增资料时的记录方式

新增资料后，建议在 `project_index.md` 中记录：

```text
- 日期：
- 新增资料：
- 存放位置：
- 资料类型：
- 核心信息：
- 是否替代旧资料：
- 待确认事项：
```

示例：

```text
- 日期：2026-05-29
- 新增资料：Find X10e 竞品价格配置对比 v1
- 存放位置：03_competitors/2026-05-29_Find_X10e_竞品价格配置对比_v1.xlsx
- 资料类型：竞品分析
- 核心信息：补充了 A 品牌与 B 品牌在价格、内存、影像、续航上的对比。
- 是否替代旧资料：否
- 待确认事项：竞品价格需在发布前再次核对。
```
