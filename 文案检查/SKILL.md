# 输出件规范与合规审查

## 目标

生成可复核的审查副本：在问题原位置添加批注，并向用户汇报问题数量、风险等级和需补充的证据。此技能是内部风险筛查流程，不替代法务终审或官方产品参数确认。

## 强制约束

- 始终保留输入原件；输出名称添加 `_审查批注版` 后缀，禁止覆盖输入文件。
- 只做批注和审查汇总，禁止直接替换正文、改写单元格值、修正公式、接受修订或删除原批注。
- 将推荐写法写入批注，不将推荐写法自动写回正文。
- 无法从规范或官方口径确认的问题，标注为 `待确认`，不得臆断为错误。
- 涉及广告合规的结论写成风险提示；如用户需要正式对外发布或法律结论，应提示需由合规或法务复核。

## 用户引导

启用技能后，在开始读取和审查文件前，先用 2-4 句中文简短说明本次会做什么。说明应包含：

- 会检查书写规范、错别字/明显异常、参数写法和广告合规风险。
- 只生成标注副本，不直接改正文或原始单元格内容。
- Excel 输出会包含可见色标、简洁批注、审查记录页和问题位置跳转；Word 输出会使用原文位置批注。
- 审查结果是内部预审提示，不替代最终参数确认或法务复核。

可使用这类话术：

```text
我会用输出件规范与合规审查流程处理这份文件：检查文字规范、参数写法、疑似错别字和广告合规风险。原文件不会被覆盖，我会生成一个标注副本；Excel 会用颜色标出问题单元格，并在审查记录页提供可点击跳转。合规类问题会按风险提示处理，正式发布前仍建议由参数负责人或法务确认。
```

## 读取规则

开始审查前读取以下参考文件：

- `references/writing-parameter-rules.md`：产品基础信息和参数写法规范。
- `references/advertising-risk-rules.md`：用户提供的广告绝对化用语筛查清单。

若用户另行提供更高优先级的品牌词典、官网参数页、最终参数表或法务标准，以该材料为准，并在审查摘要中列明所用依据。现有参考文件未提供具体型号字或命名规则细则，不自行扩展这类判断。

## 审查流程

1. 确认文件类型、输出触点和是否另有官方口径材料；未提供口径材料时仍继续做可确定的文字和风险检查。
2. 创建原件的工作副本，仅在副本上添加批注。
3. 提取可见文本、表格内容、标题、页眉页脚或工作表有效区域；保留位置映射，以便批注准确落点。
4. 分四类逐项检查：
   - `广告合规`：绝对禁用词、需要证据和限定范围的慎用词，以及首发、独家自研、全球最薄、满级能力等需证明的事实宣称。
   - `参数规范`：单位大小写、符号、专有名称、适用条件、来源一致性。
   - `文字规范`：名称口径一致、明显错别字、标点和空格异常、前后表述冲突。
   - `待确认`：需要官网参数页、品牌词典、测试报告或法务判断才能确定的事项，以及拟交付内容中尚未清除的 `xx`、`xxxx`、`TBD` 或占位提示。
5. 对每一个问题在原位置添加批注；相同问题在多个独立发布位置出现时均需定位，汇总时可合并统计。
6. 输出简短审查摘要和带批注副本，不输出一份已改写正文的“修正版”。

## 风险等级

| 等级 | 使用场景 | 批注动作 |
| --- | --- | --- |
| `高风险` | 命中内部绝对化清单，且未见官方指南中的语境排除或证明材料 | 明确提示发布前处理并交合规复核，不直接宣称违法 |
| `中风险` | 慎用词缺少证明或限定范围；效果、排名、数据缺少来源 | 请求补充证明、范围、时间或条件 |
| `规范问题` | 参数/专名/符号/错别字/一致性问题 | 写明依据和建议写法 |
| `待确认` | 证据不足或依赖最终参数口径 | 说明所需材料，不判定为错误 |

## 批注格式

原位置批注的作用是让用户立刻知道哪里需要处理，保持一句话即可。详细依据、处理建议和所需材料放入审查记录页或汇总，不重复堆进每条批注。

```text
[高风险] “最佳”为风险表述，请调整。
```

```text
[需说明] “行业首款”请在备注中注明证明材料、范围和来源。
```

```text
[规范] 倍率符号请统一为“×”，并核对参数口径。
```

```text
[待补充] 此处含占位内容，请补齐后再审。
```

- 批注只写当前单元格的问题与最短处理方向，不加入法规说明、长段依据或与处理无关的英文。
- 对需证明的词，例如 `行业首款`、`首创`、`独家自研`、`全球最薄`、`满级防水`，写明“请在备注中注明证明材料/测试条件/范围/来源”即可。
- 详细依据和完整建议保留在审查记录中，便于集中复核。

## Word 审查

- 同时使用 `documents` 技能处理 `.docx`。
- 使用原生 Word 批注锚定具体词句；优先调用 documents 技能包中的 `scripts/comments_add.py` 或 OOXML comments helper。
- 批注作者写为 `规范审查`；不得使用修订模式替代批注。
- 将带批注副本保存为 `<原文件名>_审查批注版.docx`。
- 交付前结构化验证批注确实存在、关系完整且可定位。若环境可渲染，再按 documents 技能要求渲染并检查页面布局；批注不一定在渲染页中显示，因此结构验证不可省略。

## Excel 审查

- 同时使用 `Spreadsheets` 技能处理 `.xlsx`、`.xls` 或 `.csv`；CSV 需先输出为可承载批注的 `.xlsx` 副本。
- 使用 `@oai/artifact-tool` 导入并保留现有单元格内容和公式；除明确的审查标记外，不改业务样式。
- 添加批注前设置作者：`workbook.comments.setSelf({"displayName": "规范审查"})`。
- 在问题单元格上添加 threaded comment：`workbook.comments.addThread({"cell": sheet.getRange("E2")}, "批注内容")`。
- Excel/WPS/预览工具可能不显示批注提示，因此不得只依赖批注。默认同时给问题单元格增加浅色底纹：`高风险=浅红`、`中风险=浅橙`、`规范问题=浅黄`、`待确认=浅蓝`；同一单元格命中多类时按最高风险着色。
- 默认增加一个 `审查记录` 工作表，列出色标说明及 `序号 / 位置（点击跳转） / 类别 / 风险等级 / 原文摘录 / 问题 / 建议 / 依据`；位置列必须使用可见的工作簿内部跳转链接，点击后定位到对应页签的问题单元格。为避免部分预览软件不显示公式链接文本，先写入可见位置文字，再用 `scripts/add_xlsx_review_links.py` 写入内部链接。此表属于审查标注，不修改业务数据。
- 仅用底纹作为可见定位标记，不通过替换文字、修改公式或改写业务信息“修复”原表。若用户明确要求保留原视觉不加底纹，再改为仅批注和审查记录，并预先说明客户端可能看不到批注落点。
- 将副本保存为 `<原文件名>_可见标注版.xlsx`，检查关键范围、可见底纹、简洁批注、审查记录跳转链接及导出后的可读性。

## 审查摘要

向用户提供：

- 已生成的带批注副本链接。
- 发现数量，按 `高风险 / 中风险 / 规范问题 / 待确认` 分类统计。
- 需要用户补充的官方材料，例如参数表、测试条件证明或法务确认。
- 若引用的是用户提供的内部清单而非最新官方法规，明确说明其适用范围。

同时说明交付件的使用方法，保持简短：

- Excel：先看 `审查记录` 页；点击 `位置（点击跳转）` 可到问题单元格；颜色表示风险等级；单元格批注是简短处理提示。
- Word：查看正文旁批注；批注只提示问题和处理方向。
- 详细依据以审查记录或汇总为主，原位置批注保持短提示。
- 原件未覆盖，标注版只用于审阅和修改定位。

可使用这类话术：

```text
文件里新增了审查记录页，建议先从那里看总览；点击“位置（点击跳转）”可以直接到对应问题单元格。颜色用于快速区分风险等级，单元格批注只保留最短处理提示，详细依据集中在审查记录中。原文件未覆盖，标注版方便你逐项定位和修改。
```

不要在摘要中逐条重写整份交付件；问题的落点和建议以副本中的批注与审查记录为主。
