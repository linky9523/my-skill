# 产品基础信息与参数写法规则

## 来源与边界

- 来源文件：`产品策划书写规范_基础信息参数型号命名(2).docx`，用户于 2026-05-27 提供。
- 文件标题包含“型号字与命名规范”，但当前可提取正文仅给出了产品基础信息与参数书写内容；未出现可执行的型号字或命名明细。审查时不得凭空补充该部分规则。
- 适用触点：发布会、官网、商详、电商、KV、场景图、样张样片、视频及其他产品信息传播触点。

## 产品基础信息

- 产品基础信息包括产品名称、系列名称、型号字、颜色名、版本信息、系统名、功能名、卖点名、认证信息和上市信息等。
- 不应因传播触点不同而随意改写基础信息。发现同一交付件中表述不一致时，添加一致性批注并要求对照统一口径。

## 参数总原则

- 参数以准确、统一、可核查为优先。
- 外部材料优先参考官网参数页；内部材料可参考产品信息总表，但出街前须与最终参数口径对齐。
- 尺寸、重量、厚度、亮度、功率、续航等数值不得擅自调整小数位或保留位数。
- 涉及充电、防水、续航、耐摔、亮度等效果表达，应明确测试条件或适用范围。

## 单位、符号与名称

| 项目 | 推荐写法 | 可直接标注的问题 |
| --- | --- | --- |
| 存储容量 | `GB` / `TB` | `gb`、`Gb` 等大小写偏差；组合口径仍须对照确认 |
| 电池容量 | `mAh` | `mah`、`MAH` |
| 充电功率 | `W` | 单位大小写问题；带充电时长但无测试条件 |
| 屏幕刷新率 | `Hz` | `hz`；与视频帧率混用 |
| 视频帧率 | `fps`，如 `4K 60fps` | 视频写作 `4K 60Hz` |
| 倍数表达 | `×`，如 `10×`、`120×` | 以英文字母 `X` 表示倍数 |
| 光圈值 | 正文可用 `F1.8`；参数页使用 `ƒ/1.8` | `f1.8`、`F/1.8`、`ƒ1.8`；须结合所在页面类型 |
| 无线连接 | `Wi-Fi` | `Wifi`、`WIFI`、`WiFi`，除非官方另有口径 |
| 系统名称 | `ColorOS` | `Color OS`、`colorOS` |
| 尺寸/厚度 | `mm` | 数值、小数位或“约”字与参数源不一致 |
| 重量 | `g` | “约”字、版本/颜色/工艺影响未与参数源对齐 |
| 像素 | `万像素` / `MP` | 中文材料中混用未经确认的中英文口径 |

## 高频标注样例

| 原文 | 标注建议 |
| --- | --- |
| `5000mah` / `120hz` / `12gb` | 建议写作 `5000mAh` / `120Hz` / `12GB` |
| `10X 光学变焦` / `120X 数字变焦` | 建议使用乘号：`10× 光学变焦` / `120× 数字变焦` |
| `Color OS` / `colorOS` / `WIFI` | 建议写作 `ColorOS` / `Wi-Fi` |
| `4K 60Hz 视频` | 视频帧率建议写作 `4K 60fps 视频` |
| `30分钟充满` | 标注缺少测试条件或最终确认表达，要求核验而非代改 |

## 审查前核验材料

- 官网参数页、产品信息总表或最终确认参数表。
- 数值、小数位、约值、单位大小写的统一口径。
- 充电、防水、续航、耐摔等能力声明的测试条件和适用边界。

## 交付完整性提示

- 拟交付的文案或参数内容中出现 `xx`、`xxxx`、`TBD`、`（占位）` 等未定内容时，标注为 `待确认`。
- 此类标注不判断文字正误，只提醒对外交付前补齐内容，并对补齐内容重新执行参数和广告风险检查。
