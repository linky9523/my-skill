# OPPO Brand Visual Guidance For Product Info House

Use this reference when creating OPPO-facing product information house Image2 visual references and editable PPTX files, or when the user does not provide a color/style direction.

## Default Theme

- Default to a green and white visual system.
- Primary color: OPPO green.
- Background: clean white or very light warm white.
- Supporting colors: deep green, soft green tint, neutral dark gray, light gray lines.
- Use red only for risk/todo labels, and keep it restrained.

## Visual Tone

- Clean, premium, calm, and product-strategy oriented.
- Avoid cartoon, overly decorative, neon, cyberpunk, heavy gradients, or colorful magazine style.
- The deliverable should feel suitable for internal product planning, launch alignment, white papers, and strategy decks.
- Use clear hierarchy and enough whitespace.

## Product Information House Layout

- Horizontal information house diagram by default.
- Follow the user's confirmed hierarchy first.
- If the user has no hierarchy, use the default product-house layout:
  - Roof: product positioning.
  - Core row: core user value.
  - Three pillars: user value, product capability, communication expression.
  - Foundation: evidence and optional risk/todo.
- Keep the house shape recognizable; do not turn it into a generic card grid.

## Typography And Text

- Chinese text must be legible and not crowded.
- Use concise headings and short bullet points.
- Do not add new product claims, numbers, certifications, or awards beyond the provided content.
- Avoid extreme claims such as "最强", "第一", "唯一", "行业领先" unless the source material explicitly supports them.

## Editable PPTX Requirements

- Generate and obtain approval on an Image2 visual reference before reconstructing the editable PPTX.
- Default to a 16:9 single-slide information house graphic.
- Keep all major copy as editable PowerPoint text boxes and use editable vector shapes for the roof, dividers, pillars, foundation, and emphasis lines.
- Keep the OPPO green and white theme easy to modify: a white canvas, green roof/key lines, neutral body text, and soft-green supporting fills.
- Do not flatten the complete slide into an image. Supporting product or scenario photos may be raster images when they improve the deliverable.
- Use the approved Image2 reference as the visual target for composition, hierarchy, spacing rhythm, and tone; a technically editable but visibly inferior reconstruction is not acceptable.
- Render and review the slide before delivery to catch overflow, small text, overlaps, or hierarchy confusion.

## Image2 Reference Prompt Requirements

When Image2 is called for the pre-PPT visual reference or final image-only output, include:

```text
Create a horizontal product information house diagram in OPPO-inspired green and white visual style.
Use a clean white background, OPPO green roof and key lines, dark gray body text, restrained red only for risk/todo.
The layout must include: roof positioning, core value row, three pillars, foundation evidence, foundation risk/todo.
Keep all Chinese text legible and faithful to the provided content.
Do not invent product claims, parameters, certifications, awards, or extreme statements.
Make it look like a premium product strategy deliverable, not a cartoon illustration.
```

If the user provides a custom hierarchy, replace the default roof/core/three-pillar/foundation instruction with the user's confirmed hierarchy. If the user explicitly says no risk layer is needed, do not add a risk/todo section to the PPTX or image.

If the user provides a different brand or color direction, use the user's direction instead of this default.
