# Official Website Remark Patterns

## Core Style

- Use sober compliance wording, not persuasive copy.
- Start from the exact claim: parameter, function, sample, material, accessory, test result, or availability condition.
- Name the exact claim object in the first phrase whenever possible. Prefer `主摄镜头进光量数据...`, `AI 一键闪记...`, `卫星通信功能...`, `4K 超清实况功能...`, or `相关配件或套装...` over generic phrases.
- Keep one row's remark tied to that row only. Do not add broad all-page disclaimers into every row.
- Within the same `屏数`, do not repeat identical remark lines. Keep the first instance and use `/` on later rows unless they add a new condition.
- Use placeholders such as `xxx`, `xx`, or product names only when the source row does not provide confirmed values.
- If a row already has a valid remark, keep it and only add missing constraints when needed.
- Use `/` when the row is only a planning label, section divider, ordinary public claim, or internal material note with no standalone legal/compliance risk.
- Keep broad statements such as "data comes from OPPO laboratory" in the final global note. Do not repeat that line row by row unless the row makes a specific comparison, uplift, certification, or condition-sensitive claim.
- Do not use vague fallback remarks like `相关对比、提升或行业表述基于特定测试条件或统计口径`.
- Never write `xxx 为页面卖点表述`. If a row only has broad superlatives such as `最高`, `首款`, `行业领先`, or `独家` but no concrete condition or comparison object, use `/`; the claim should be reviewed as copy risk rather than padded with a remark.
- Pure objective values such as transfer speed, refresh rate, brightness, capacity, dimensions, or `最高达 xxx` are usually covered by the final global lab-data note and do not need row-level remarks.
- Chip names alone are not battery claims. `电池管理芯片`, `冰川电池管理芯片`, or chipset lists should not trigger battery remarks unless the row also claims capacity, charging, battery life, cycle life, typical/rated values, or low-temperature battery performance. Avoid substring false positives such as `抢票成功率`, which contains `功率` but is not a charging-power claim.

## Write Only For Real Risks

Most rows do not need a generated remark. Write one only when the row includes at least one of:

- Existing human remark or explicit footnote marker such as `（12）`.
- Specific comparison or uplift with a concrete measured object and comparison scope, such as lens light intake, pixel uplift, GPU power, memory usage, or battery cycle life.
- Feature availability restriction: only some models, apps, regions, operators, accounts, versions, OTA, switches, or networks.
- Accessory or kit sold separately.
- Certification, medical-adjacent, eye-care, waterproof/dustproof/drop-proof, satellite/operator, AI-generated, or third-party service claim.
- A sample photo/video claim that may be mistaken for guaranteed real-world output.

## Common Categories

### Specific Claim Patterns

- Comparison/uplift with a measurable object:
  `主摄镜头进光量数据来自 OPPO 实验室，相关提升或对比结果基于页面所述测试对象及测试条件，因测试环境、方法、软件版本、设备状态等不同，数据可能存在差异，请以产品实际为准。`
- Broad industry/first/only/leading claim with no concrete condition:
  Use `/`. Do not generate a remark just to restate that it is a selling point.
- Mode-limited camera claim:
  `4K 超清实况功能需在指定拍摄模式下开启对应功能开关，实际效果可能因拍摄环境、方法、设备状态等不同而存在差异，请以产品实际为准。`
- App/version-limited software claim:
  `AI 一键闪记相关参数或部分功能体验仅部分应用支持，具体支持的应用、功能和界面，请以实际为准。AI 生成内容仅供参考。`
- Sold-separately accessory:
  `需要单独购买，详细信息以官方网站介绍为准。`

### General Product And Design

Use mainly in the final global note, not for ordinary appearance rows:

`产品图片仅供参考，请以实物为准。产品规格、外观细节、材质及工艺可能因生产批次、供应商、测量方法等因素略有差异，请以产品实际为准。`

### Display And Eye Care

Use when the row contains eye-care certification, medical-adjacent language, or a footnoted brightness/refresh-rate claim:

`屏幕相关数据来源于 OPPO 实验室，实际显示效果可能因使用环境、设备状态、软件版本、测试方法及个人使用习惯不同而存在差异，请以实际体验为准。护眼相关功能非医疗器械功能，不具有治疗作用。`

### Camera Hardware And Photo Effects

Use when the row makes mode-limited, comparison, uplift, superlative, or footnoted camera claims:

`影像规格、焦段、进光量、动态范围等数据来源于 OPPO 实验室或供应商测试。实际拍摄效果可能因拍摄环境、拍摄对象、设备状态、软件版本及拍摄模式不同而存在差异，请以实际体验为准。`

### Samples And Materials

Use when sample photos/videos may be mistaken for guaranteed actual output:

`页面样张、视频、产品界面、结构示意图及素材效果仅供参考，可能经过裁切、压缩、合成或创意化呈现，请以实际拍摄效果、产品界面和最终上线页面为准。`

### Zoom

Use when the row discusses optical zoom, optical-quality zoom, digital zoom, telephoto, equivalent focal length, teleconverter, or long-distance shooting:

`变焦、等效焦段及远摄效果可能因拍摄距离、光线、模式、算法处理及设备状态不同而存在差异。光学品质变焦、数字变焦等能力请以产品实际支持情况为准。`

### Video, Log, HDR, And Live Photo

Use when the row discusses 4K, 8K, HDR, Log, LUT, live photo, motion photo, stabilization, or video color:

`视频、HDR、Log、LUT、实况照片等功能的可用规格和效果与拍摄模式、软件版本、存储空间、环境光及设备状态有关，请以实际体验为准。`

### AI And Software

Use when the row discusses AI, ColorOS, UI, albums, assistant features, recognition, editing, or system smoothness:

`以上软件界面和功能仅作展示。AI 及系统相关能力可能因软件版本、地区、账号登录状态、网络环境、适配应用及个人使用习惯不同而存在差异，请以实际版本功能及界面为准。`

### Battery And Charging

Use when the row discusses capacity, battery life, cycle life, wired charging, wireless charging, reverse charging, or low-temperature charging:

`电池容量、续航和充电数据来源于 OPPO 实验室或供应商测试，实际表现可能因网络环境、使用方式、充电条件、设备温度、电池老化及其他因素不同而存在差异，请以实际体验为准。`

### Waterproofing And Durability

Use when the row discusses IP ratings, waterproofing, dust resistance, drop resistance, scratch resistance, or harsh tests:

`防水、防尘、抗摔、抗划等数据来源于受控实验室条件。相关防护能力并非永久有效，可能因日常磨损而下降；请勿在潮湿状态下充电，因进液、跌落、碰撞等导致的损坏请以保修政策为准。`

### Performance And Storage

Use when the row discusses chipsets, performance uplift, benchmark scores, RAM, storage, smoothness, thermal data, or gaming:

`性能、跑分、存储、散热及游戏表现数据来源于 OPPO 实验室或官方测试，实际表现可能因测试环境、软件版本、内存规格、网络环境、应用适配及个人使用习惯不同而存在差异。`

### Connectivity, Sound, And Third Parties

Use when the row discusses cellular network, satellite/offline communication, NFC, cross-device features, volume, audio, third-party apps, or third-party content:

`通信、音频、协同及第三方服务的功能表现可能因地区、运营商、网络环境、设备状态、软件版本、应用适配及第三方服务政策不同而存在差异，请以实际支持情况为准。`
