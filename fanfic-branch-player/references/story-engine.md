# 互动同人剧情引擎

## 角色一致性

先为每个关键角色确认四件事：

| 项目 | 作用 |
| --- | --- |
| 表层目标 | 角色此刻主动追求什么 |
| 深层需求 | 角色真正缺少但未必承认什么 |
| 核心恐惧 | 什么会让角色退缩、攻击或自我欺骗 |
| 关系底线 | 角色愿意为谁做到什么程度，又绝不会做什么 |

改变事件不等于改变人格。角色可以在新信息、新代价或关系推动下做出原作中没有的选择，但要让读者看到促成变化的原因。

当用户解释与模型记忆冲突时，采用用户解释，并更新 `原作基线.md`。

## 蝴蝶效应

每个新选择至少推演三层：

1. **立即结果**：本场景立刻发生什么。
2. **关系或信息变化**：谁更信任谁，谁知道了新事实，谁产生误解。
3. **延迟代价**：这个好结果以后会制造什么新风险。

理想结局也应有阻力。避免通过角色突然坦白、反派突然失误或规则突然失效，无成本解决原作冲突。

## 三选项设计

三个选项应采用不同的“行动机制”，例如：

- 公开真相：增加合作可能，同时失去秘密优势。
- 私下结盟：保护关系，同时制造对其他人的隐瞒。
- 主动承担风险：快速推进目标，同时让角色暴露或受伤。

至少满足以下三项中的两项：

- 直接代价不同。
- 受益或受损角色不同。
- 打开的下一类场景不同。

不合格示例：

- A：立刻追上去。
- B：谨慎地追上去。
- C：想清楚后追上去。

合格示例：

- A：当众拦下对方，迫使所有人见证这次对话。
- B：放对方离开，转而调查导致误会的证据。
- C：请共同朋友传递一句只有两人听得懂的话。

## 场景节奏

800–1500 字场景优先采用：

1. 进入正在发生的动作或压力。
2. 用细节快速定位人物情绪与关系。
3. 让一个原作惯性造成阻碍。
4. 通过行动或对白改变状态。
5. 在不可同时兼得的选择上收束。

不要在每轮重新介绍全部背景。只提醒当前场景必须知道的信息。

## 结局判断

满足以下条件时，可以建议进入结局：

- 改写目标已经成功、失败或转化为新的明确答案。
- 核心关系完成一次不可逆变化。
- 主要秘密或冲突已被处理。
- 最近选择的代价已经兑现，而非被跳过。

结局可以保留余味，但不能把所有冲突都留给“下一章”。后日谈用于展示新常态，不用于补做尚未解决的主冲突。
